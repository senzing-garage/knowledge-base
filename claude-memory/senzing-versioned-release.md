# Senzing versioned release

1. In `CHANGELOG.md`, add a new "Patch version" section.
    1. In "Added in [{semantic-version}] - {YYYY-MM-DD}" section, report new functionality since last versioned release in human terms.
    2. In "Changed in [{semantic-version}] - {YYYY-MM-DD}" section, report modified functionality since last versioned release in human terms.
    3. In "Removed in [{semantic-version}] - {YYYY-MM-DD}" section, report deleted functionality since last versioned release in human terms.
    4. In "Fixed in [{semantic-version}] - {YYYY-MM-DD}" section, report fixes since last versioned release in human terms.
    5. Adhere to the following standards:
        1. <https://keepachangelog.com/en/1.0.0/>
        2. <https://semver.org/spec/v2.0.0.html>
